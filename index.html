PonsWarp

초고속 대용량 P2P 파일 전송

Performance
Challenge

브라우저 메모리 한계로 인한 대용량 파일 전송 중단 및 SCTP 프로토콜 상의 전송 속도 불안정.

Solution
  • 64kb 청크 사이즈가 가장 안정적임을 실험으로 도출 및 적용.
  • OPFS(Origin Private File System) 도입으로 메모리를 거치지 않는 디스크 직접 쓰기 구현.
  • 향후 dCDN 도입을 위한 아키텍처 설계.
